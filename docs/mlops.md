# MLOps Setup

## Overview

This project uses **MLflow** for local experiment tracking. Every model training
run is automatically registered with its parameters, metrics, artifacts, model
object, signature, and input example.

MLflow tracking is configured in `src/mlops/mlflow_tracking.py` and called
from `scripts/03_train_models.py` and `scripts/05_run_full_pipeline.py`.

---

## Configuration

All MLflow settings are defined in `src/config/settings.py`:

| Parameter                        | Value                                  |
|----------------------------------|----------------------------------------|
| `MLFLOW_TRACKING_URI`            | `http://127.0.0.1:5000`                |
| `MLFLOW_EXPERIMENT_NAME`         | `monthly-investment-recommendation`    |
| `LOGISTIC_REGRESSION_RUN_NAME`   | `logistic-regression-classifier`       |
| `RANDOM_FOREST_RUN_NAME`         | `random-forest-classifier`             |
| `LINEAR_REGRESSION_RUN_NAME`     | `linear-regression-expected-returns`   |
| `PORTFOLIO_OPTIMIZATION_RUN_NAME`| `portfolio-optimization`               |

## MLflow Setup

### Step 1 — Start the MLflow server

Open a terminal and run:

```bash
mlflow ui --host 127.0.0.1 --port 5000
```

> Keep this terminal open while the pipeline runs.

### Step 2 — Open the UI
http://127.0.0.1:5000

### Step 3 — Run the pipeline in a second terminal

```bash
python scripts/05_run_full_pipeline.py --amount 10000
```

## What Gets Logged Per Run

Each call to `log_model_run()` registers one MLflow run with:

| Category       | Details                                                      |
|----------------|--------------------------------------------------------------|
| Parameters     | Model hyperparameters from `settings.py`                     |
| Metrics        | All evaluation metrics (CV + test for classifiers, MAE/RMSE/R2 for regression) |
| Model          | Serialized scikit-learn pipeline via `mlflow.sklearn`        |
| Model signature| Input/output schema inferred from training data              |
| Input example  | First 5 rows of the feature matrix                           |
| Artifacts      | Confusion matrix and ROC curve plots (classification only)   |

|-----------------------------------|----------------|---------------------------|

## MLflow Directory Structure

MLflow stores all data locally in `mlruns/` at the project root:

```text
mlruns/
└── 1/                             ← Experiment ID
    ├── meta.yaml                  ← Experiment metadata
    └── <run_id>/
        ├── artifacts/
        │   ├── model/             ← Serialized sklearn pipeline
        │   ├── confusion_matrix_<model>.png
        │   └── roc_curve_<model>.png
        ├── metrics/               ← One file per metric
        └── params/                ← One file per parameter
```

## Experiment Tracking Flow
configure_mlflow()
│
▼
Set tracking URI + create/get experiment
│
▼
For each model:
log_model_run(
run_name,
model,
model_type,
metrics,
params,
artifacts,
input_example,
)
│
▼
mlflow.start_run(run_name=run_name)
├── mlflow.log_params(params)
├── mlflow.log_metrics(metrics)
├── mlflow.sklearn.log_model(model, signature, input_example)
└── mlflow.log_artifact(confusion_matrix_path)
mlflow.log_artifact(roc_curve_path)

## Metrics Summary

In addition to MLflow, all metrics are saved locally to:

```text
reports/metrics/model_metrics.json
```

This file is generated by `save_metrics_to_json()` in
`src/models/evaluate_models.py` and contains the consolidated
results of all three models in a single JSON.

## Environment Variables

The project uses `python-dotenv` for environment configuration.
Create a `.env` file at the project root if needed:

```bash
# .env (example)
MLFLOW_TRACKING_URI=http://127.0.0.1:5000
```

> `.env` is listed in `.gitignore` and must never be committed.
> Use `.env.example` as the reference template.
